# План автоматизации тестирования
## 1. Перечень автоматизируемых сценариев

В рамках дипломного проекта будет автоматизировано тестирование функционала приобретения тура через веб-интерфейс приложения aqa-shop.

#### 1. Позитивные сценарии (Happy Path)
##### Сценарий №1

Название: Успешная покупка тура по карте

Данные: валидные данные карты

Шаги:

Открыть страницу приложения
Нажать кнопку "Купить"
Ввести валидные данные карты
Нажать кнопку "Продолжить"

Ожидаемый результат:

* появляется сообщение об успешной оплате
* в базе данных создаётся запись со статусом APPROVED

##### Сценарий №2

Название: Успешное приобретение тура в кредит

Данные: валидные данные карты

Шаги:

Открыть страницу приложения
Нажать кнопку "Купить в кредит"
Ввести валидные данные карты
Нажать кнопку "Продолжить"

Ожидаемый результат:

* появляется сообщение об успешном оформлении кредита
* в базе данных создаётся запись со статусом APPROVED

## 3. Возможные риски при автоматизации
### Технические риски
1. проблемы с запуском Docker
2. проблемы с подключением к PostgreSQL
3. нестабильные элементы интерфейса
### Риски автоматизации
1. нестабильные тесты (flaky tests)
2. ошибки в локаторах
3. ограниченное время на исправление ошибок
